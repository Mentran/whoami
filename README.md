# 我是谁

一个宝可梦粉丝向的「Who's That Pokemon?」复古掌机猜影子小游戏。

## 产品目标

打开页面后，用户选择难度并开始挑战：看到一只宝可梦黑影后直接说出名字，答对后揭晓真身并计分。当前版本专注于语音主持、复古掌机氛围、真实图鉴资料和稳定可玩，不做后端或登录。

## MVP 范围

- 复古红色掌机界面
- 前三世代 386 只宝可梦题库
- 黑影猜谜 + 10 秒倒计时
- 模式选择：10 题挑战 / 无限模式
- 中文名 / 英文名 / 常见别名判定
- 难度规则：简单只抽第一世代；普通 / 困难抽前三世代全题库；困难不接受别名和接近答案提示
- 语音答题：猜错可继续，超时 / 跳过后揭晓答案
- 文字答题兜底：麦克风不可用时也能完成挑战
- 语音指令：下一题、跳过、介绍一下、重新开始、静音、打开声音
- “我是谁”小百科：本题结束后可语音触发，浏览器 TTS 自动朗读
- 连击反馈：连续答对显示 COMBO，结算页统计最长连击
- 结果反馈：平均答题时间、无限模式徽章和 title
- 5 首 8-bit 风格背景音乐：默认开启，只在开始页和结算页播放，支持开关、切歌和音量
- HIT / TOT 计分、10 题挑战和本地最高分
- 结算页生成 PNG 分享图
- 桌面和移动端适配

## 技术方案

- Vite
- React
- TypeScript
- 原生 CSS
- 本地题库 + 本地缓存的 386 张 PokeAPI official artwork 图片
- 图鉴资料优先使用 PokeAPI 中文文本，缺失时回退官方中国图鉴页的中文描述
- Web Speech API 语音识别
- speechSynthesis 语音合成
- `localStorage` 保存最高分和每只宝可梦的本地答题记录

## 本地运行

```bash
npm install
npm run dev -- --host 127.0.0.1 --port 5175
```

访问 `http://127.0.0.1:5175/`，点击“开始挑战”后允许麦克风权限。

## 验证

```bash
npm test
npm run build
```

## 浏览器兼容

- 推荐：最新版 Chrome / Edge。
- Safari、Firefox 或部分移动浏览器可能不支持 Web Speech API，页面会提示切换浏览器。
- 如果出现“麦克风权限被拒绝”，请点击地址栏麦克风图标，将权限改为允许后刷新页面。
- 开发环境会显示一个调试输入框，用于绕过麦克风验证答案和指令；正式构建不会显示。

## 里程碑

### M1：可玩闭环

- 初始化前端项目结构
- 实现掌机外壳和游戏屏幕
- 内置一批宝可梦题目
- 完成输入、判题、揭晓、计分

### M2：复古质感

- CRT 扫描线
- 答错震动
- 答对闪屏
- Reveal 动画
- 按钮触感

### M3：内容扩展

- 扩展到前三世代 386 只
- 增加难度和连续挑战
- 支持更多别名

### M4：传播增强

- 分享结果图
- 每日挑战
- 自定义题库草案

### M5+：语音主持模式

- 语音输入答题
- 每题倒计时
- 语音控制下一题 / 跳过 / 小百科
- 失败后自动播报正确答案
- 小百科朗读期间暂停语音识别
