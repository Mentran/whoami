# 下一步开发规划

## 阶段判断

当前项目已达到 V1 内测可玩水平：核心闭环、前三世代 386 题库、语音输入、TTS 小百科、计分、PNG 分享图和基础测试都已完成。

下一阶段先把真实浏览器体验和内容质量收口，再做能提高复玩的玩法增强。

## 北极星目标

- 开局成功：玩家打开页面后 30 秒内能进入第一题。
- 完局成功：玩家即使没有麦克风权限，也能完成 10 题挑战。
- 复玩动机：结算页能让玩家清楚知道自己表现如何，并愿意再试一次。
- 传播准备：每日挑战和分享图上线前，先保证单局体验稳定。

## 开发原则

- 继续保持纯前端，不引入后端、数据库、登录系统。
- 优先修影响完局的体验问题，再做增强玩法。
- 每次功能推进后同步更新 `TODO.md`。
- 新增逻辑优先补测试；视觉和语音流程至少做一次桌面 + 移动手动验证。

已完成：

- 第二、第三世代百科在 PokeAPI 中文不足时回退官方中国图鉴，已清理编号兜底内容。
- 连击系统第一版：局内反馈、最长连击统计、结算展示。

待继续：

验收：

## 初步方案

### 1. 先做 V1 验收

- 按 `QA_CHECKLIST.md` 复跑语音、TTS、文字兜底、移动布局和分享流程。
- 测试矩阵先控制在桌面 Chrome、桌面 Edge、移动 Chrome，避免一开始扩到低优先级浏览器。
- 只要出现无法开局、无法下一题、识别 / 播报互相卡死，优先修稳定性，不进入新玩法开发。

### 2. 再做百科质量治理

- 先审计数据而不是直接手工改：找出第二、第三世代描述过短、重复、缺少生态信息的条目。
- 优先修 `scripts` 生成策略、去重和文本选择规则，让数据可再生成。
- 对少量 API 数据不足的条目再加人工兜底，验收标准是“资料介绍”，不是“适不适合出题”。

### 3. 再做玩法增强

- 连击第一版已保持基础分数不变；后续再结合难度调优判断是否补音效变化或轻量奖励。
- 平均答题时间只统计答对题，跳过和超时不计入速度；失败表现继续由命中率或无限模式结束条件表达。
- 无限模式作为独立模式接入首页：答对继续，超时 / 跳过结束，按连续答对数给徽章和 title；暂不做排行榜和专属分享图。
- 提示系统放在连击之后接入，提示出现时机由剩余时间和难度决定。
- 难度调优最后统一处理，避免连击、提示和别名规则互相推翻。

### 4. 本地答题记录与自适应出题

- 本地答题记录已先落地，不单独造“错题集”实体；记录层按宝可梦聚合见过次数、命中、失误、连续答对、最近出题时间。
- 简单难度仍可作为熟悉第一世代的随机热身；普通 / 困难默认优先未见题和薄弱题。
- “已掌握”先按至少 2 次出题且连续全对判定。普通 / 困难抽题时优先避开它们，但题池不足、固定挑战和复习入口仍允许回流。
- 记录层先保持独立，后续错题复习、本地图鉴收集和题目调度都消费它，避免多套进度互相打架。

## 最近 3 个任务

1. 基于本地答题记录设计普通 / 困难的自适应抽题权重。
2. 设计并实现提示系统第一版，先控制提示种类和触发时机。
3. 统一调整普通 / 困难 / 无限模式的倒计时、提示和结算分享策略。
