# 下一步开发规划

## 阶段判断

当前项目已达到 V1 内测可玩水平：核心闭环、前三世代 386 题库、语音输入、TTS 小百科、计分、结算分享和基础测试都已完成。

下一阶段的目标不是增加更多系统，而是让真实玩家能顺利开始、顺利玩完、愿意再来一局。

## 北极星目标

- 开局成功：玩家打开页面后 30 秒内能进入第一题。
- 完局成功：玩家即使没有麦克风权限，也能完成 10 题挑战。
- 复玩动机：结算页能让玩家清楚知道自己表现如何，并愿意再试一次。
- 传播准备：每日挑战和分享图上线前，先保证单局体验稳定。

## 开发原则

- 继续保持纯前端，不引入后端、数据库、登录系统。
- 优先修影响完局的体验问题，再做增强玩法。
- 每次功能推进后同步更新 `TODO.md`。
- 新增逻辑优先补测试；视觉和语音流程至少做一次桌面 + 移动手动验证。

## 里程碑

### P0：V1 内测稳定性

目标：任何玩家都能完成一局。

- 生产环境增加文本答题兜底。
- 麦克风不可用时，界面明确切到输入模式。
- 修复移动端首屏裁切。
- 区分 TTS 文案：答案播报 / 小百科朗读 / 等待语音。
- 建立浏览器测试清单。

验收：

- Chrome 桌面允许麦克风：语音完整玩完 1 局。
- Chrome 桌面拒绝麦克风：文本完整玩完 1 局。
- 390x844 手机尺寸：首屏顶部不被裁切，游戏中无横向滚动。
- `npm test` 和 `npm run build` 通过。

### P1：好玩性增强

目标：让玩家有“再来一局”的理由。

- 连击系统。
- 提示系统。
- 难度规则重做。
- 结算页增加命中率、最长连击、最快答题、中文称号。

验收：

- 普通玩家能感知简单 / 普通 / 困难的差异。
- 答对连续题时有明确反馈。
- 结算页能解释玩家本局表现。

### P2：传播闭环

目标：让玩家愿意分享成绩。

- 每日挑战。
- 分享结果图。
- 本地图鉴收集。

验收：

- 同一天打开每日挑战题目一致。
- 分享图能表达分数、难度、日期和称号。
- 不依赖服务器即可运行。

## 最近 3 个任务

1. 建立真实浏览器测试清单并手动跑 Chrome / Edge / 移动 Chrome。
2. 桌面和移动端各完整玩完 1 局，记录语音授权、文字兜底、下一题和小百科流程。
3. 继续补齐第二、第三世代的中文百科质量，再开始 P1 连击系统方案设计。

## 暂不做

- 登录账号。
- 排行榜。
- 后端每日题库。
- 付费、商业化和正式品牌包装。
- 第四世代及之后题库扩展。
