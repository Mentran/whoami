# QA 验收清单

用于 V1 内测前的手动回归。先确认玩家能开局、能完局，再继续做新玩法。

## 1. 预检

- [ ] `npm test` 通过
- [ ] `npm run build` 通过
- [ ] 本地服务可访问：`http://127.0.0.1:5175/`
- [ ] 首屏能看到难度选择、开始按钮、A/B 操作区、背景音乐控制
- [ ] 控制台没有阻塞交互的报错

## 2. 测试矩阵

| 场景 | 浏览器 | 视口 / 设备 | 重点 |
| --- | --- | --- | --- |
| 桌面主流程 | 最新 Chrome | 桌面 | 语音授权、答题、TTS、分享 |
| 桌面兜底 | 最新 Chrome | 桌面 | 拒绝麦克风后的文字答题 |
| 桌面兼容 | 最新 Edge | 桌面 | 语音主流程与权限恢复 |
| 移动主流程 | 移动 Chrome | 390x844 或真机 | 首屏、点击区域、文字溢出、权限提示 |

## 3. 桌面 Chrome：允许麦克风

- [ ] 选择一个难度后点击开始，浏览器弹出麦克风授权请求
- [ ] 允许后进入第一题，页面显示监听状态
- [ ] 说错名字后题目不结束，倒计时继续，识别继续等待下一次答案
- [ ] 说对名字后立即揭晓真身，并有答对语音反馈
- [ ] 答对后说“介绍一下”，开始播报百科，播报期间不继续识别系统播报内容
- [ ] 百科未播完时点击 A 或“下一题”，播报停止且进入下一题，不残留“百科朗读失败”
- [ ] 超时后自动揭晓并播报正确名字
- [ ] 超时后说“下一题”，可进入下一题
- [ ] 超时后说“介绍一下”，可播报当前宝可梦百科
- [ ] 说“重新开始”或点击“重新开始”，返回难度选择

## 4. 桌面 Chrome：拒绝麦克风

- [ ] 拒绝权限后，页面不进入无限重试
- [ ] 页面明确提示麦克风不可用和恢复路径
- [ ] 文字输入框可答题，可提交宝可梦名字
- [ ] 文字输入可提交“下一题”“介绍一下”等指令
- [ ] 文字模式能完整完成 10 题挑战
- [ ] 地址栏或系统设置恢复麦克风权限并刷新后，可重新进入语音流程

## 5. 语音识别专项

- [ ] 大陆常用名称、旧译或俗称样例可命中对应宝可梦
- [ ] 常见识别错字经过后处理后可命中正确名字
- [ ] 高风险近邻不会被过度纠错成另一只宝可梦
- [ ] 无语音输入时能继续等待，不因 `no-speech` 卡死
- [ ] TTS 播报和语音识别不会同时抢占输入链路

## 6. 移动端

- [ ] 390x844 首屏不裁掉掌机顶部和开始入口
- [ ] 页面没有横向滚动
- [ ] A/B 按钮、重新开始、文字输入和发送按钮可点
- [ ] 题目切换时不先闪出原图再变回黑影
- [ ] 倒计时动画连续，不出现一秒一跳的明显卡顿
- [ ] 结算页统计、难度选择和分享按钮无重叠

## 7. 结算与分享

- [ ] 一局内不出现重复题目
- [ ] 结算页显示命中率、难度、统计项和下一局入口
- [ ] 点击分享后生成 PNG 成绩卡
- [ ] 支持系统图片分享时，分享面板能正常打开
- [ ] 不支持图片分享时，能下载图片并给出文字结果兜底
- [ ] 分享图中文、分数、统计和装饰元素没有错位

## 9. 阻塞级别

- `P0`：无法开始、无法进入下一题、语音与 TTS 卡死、文字兜底不可用
- `P1`：结果错误、题目露图、分享图错位、移动端主要按钮不可点
- `P2`：文案、轻微动画、非阻塞视觉细节
